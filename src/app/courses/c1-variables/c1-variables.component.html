Variable Types


First of all, what is a variable?

In simple words, a variable is something that holds a value. Think of a simple maths function:

  • f(x) = 2x + 1

In this case, the value of x can be changed to a number, and hence the value of f(x) will change accordingly. If you set x=0, f(x) will be equal to 1. Similarly, in code, you can assign a value to a variable.

  • (define myVar 7)

In the above code, the value 7 is assigned to a variable named myVar. For the rest of the code, every time myVar is mentioned, Dr Racket will interpret it as 7, instead of a literal text "myVar"


What are variable types?

To give a short answer, a variable type refers to the type of the data it stores. If a variable stores only integers and nothing else, then we can say that the variable's type is integer. There are slight differences between programming languages on how specific those distinctions are, but here are the basic types of variables in Typed Racket. Later in this page, I will discuss those variables further:

  • Integer: A number without decimal points. (ex: 3, 0, -71)

  • Real: A number with decimal points. (ex: 3, 3.0, 0.23, 2/3)

  • Char: A character, written after the #\ symbols. (ex: #\A, #\+, #\\, #\ )

  • String: A list of characters, written between quotation marks. (ex: "Hi", "Hello world", "", "5")

  • Image: An image.

I highly recommend checking out the official documentation about variables.


Number (Integer and Real)

A Real refers to any real number, while Integer refers to real numbers with no decimal points. Numbers do not require an additional symbol to declare, so you can directly type a number. If you are familiar with hex numbers or other bases of numbers, you can check the documention on how to use them in Racket!

Summation (+) will add every number together.

  • (+ 1) --> Returns 1

  • (+ 2.5 4) --> Returns 6.5

  • (+ 1 2 -4) --> Returns -1

Subtraction (-) will reduce all following numbers from the first number.

  • (- 1) --> Returns 1

  • (- 5 2.5) --> Returns 2.5

  • (- 5 1 1 1) --> ReturnsGives 2

Multiplication (*) will multiply all numbers with each other.

  • (* 1) --> Returns 1

  • (* 2 2.5) --> Returns 5.0

  • (* 1 2 3 4 5) --> Returns 120

Division (/) will divide the first number by all following numbers.

  • (/ 1) --> Returns 1

  • (/ 6 3.0) --> Returns 2.0

  • (/ 120 4 3 2) --> Returns 5 (because 120/(4*3*2) = 5)

Inequality signs will compare all inputs in order, and return a boolean. All of these comparisons work with 2+ numbers too!

  • (= 5 5) --> Returns #true

  • (= 3 5) --> Returns #false

  • (> 10 5) --> Returns #true

  • (> 2 5) --> Returns #false

  • (< 5 10) --> Returns #true

  • (< 5 2) --> Returns #false

  • (>= 5 5 2) --> Returns #true

  • (>= 5 6 2) --> Returns #false

  • (<= 2 2 4 4) --> Returns #true

  • (<= 2 4 2 4) --> Returns #false

It is possible to check if a number is positive, zero or negative.

  • (positive? 5) --> Returns #true

  • (zero? 5) --> Returns #false

  • (negative? 5) --> Returns #false

The following functions add or subtract 1 from a given number. You can also use the plus or minus signs, but I find these cleaner in code.

  • (add1 5) --> Returns 6

  • (sub1 5) --> Returns 4

You can use one of the following for checks. They require the testing library!

What is the difference between the two? Well, 8/2 is an integer operation with a pretty concrete answer. It is 4. For computers, those operations are also very simple. When it comes to non-integers, it is a bit difference. Imagine you are dividing 7.0 by 2.0, then you would say that the answer would be 3.5 without a doubt. Simply put, computers are weird and that is not always the solution they find. It is very likely that the answer will be something like 3.500000004 to this question. It is a close enough answer, but not exactly the correct one. Therefore, instead of checking for the correct answer, you need to check for a range of answers. You first need to set a small "tolerance" value such as 1/10000. In the below check-within example, the test will pass as long as the result is between "3.5 - 1/10000" and "3.5 + 1/10000"

  • ;; In order to check if an integer operation is correct.

  • (check-expect (/ 8 2) 4) --> Test passes!

  • ;; In order to check if a non-integer operation is correct.

  • (check-within (/ 7.0 2.0) 3.5 1/10000) --> Test passes!


String

A string is a sequence of characters; for instance, a text. In order to declare that something is a string, you need to surround it with quotation marks. For example: "Hello", "Hello world!" and "123". Don't forget that a space (" ") or an empty string ("") would make valid strings, of length 1 and 0 respectively!

As a string uses a quotation mark to mark where the string starts and ends, you cannot use a quotation mark inside a string. You can instead mark the quotation mark with a backslash (\") as an escape character. For instance: "He said "Hi!"" will not be a valid string, while "He said \"Hi!\"" will be!

The sum operation for string is called append.

  • (string-append "Hello" " world" "!") --> Returns "Hello world!"

  • (string-append "" " " "a") --> Returns " a"

You can check if two (or more) strings are the same.

  • (string=? "Hello" " world") --> Returns #false

  • (string=? "Hello" "Hello" "Hello") --> Returns #true

You can check the length of a string.

  • (string-length "Hello") --> Returns 5

  • (string-length "") --> Returns 0

You can get the character of a string at an index. Don't forget that counting starts from index 0.

  • (string-ref "ABCDEF" 2) --> Returns #\C

You can also get a part of a string (called substring) from a string. You need to specify where the substring starts and ends. The latter index is excluded!

  • (substring "ABCDEF" 2 4) --> Returns "CD"

  • (substring "ABCDEF" 1 5) --> Returns "BCDE"

  • ;; To get the rest of a string, omit the second number.

  • (substring "ABCDEF" 2 5) --> Returns "CDEF"

You can use check-expect for checks. They require the testing library!

  • (check-expect (string-append "a" "b") "ab") --> Test passes!
