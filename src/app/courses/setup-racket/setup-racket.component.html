Setting up Dr Racket


Download Dr Racket

You can use this link to download the latest version of Dr Racket.

I think it is important to make a distinction here. Dr Racket is an IDE (or the program that can interpret the programming language). Plus, IDEs often have some helpful tools like autocomplete that come with them. Without any interpreter you will not be able to run your code. Racket is the name of the programming language. More specifically, Typed Racket is the name of the programming language this guide will focus on. There is a slight difference between the two, which will be discussed later in the guide.


Install Dr Racket

Once the download is complete, run the file and follow the instructions.

It should be straightforward, but I plan on adding a FAQ section here.


Declare the language you use

Add "#lang typed/racket" at the top of your document.

This is necessary, so that the IDE knows whether to interpret Racket or Typed Racket. In that sense, you have to add this line at the top of every document you have (assuming you want to use Typed Racket). If you do not want to copy-paste this line every single time you open a project, there is a way to automatize it!

  1. Open Dr Racket.

  2. Click on “Determine language from source” in the bottom-left corner.

  3. Click on “Choose Language...”

  4. Click on ”Show Details”

  5. On bottom-right, type “#lang typed/racket” in “Automatic #lang line”

  6. Click on OK

  7. Open a new Racket project to test if it is working.


Import required libraries

Racket comes with some built-in functions which we refer as the "base" functions of this language. Of course, there are extra functionality of the language which are not included by default and/or created by a third party. Those are called libraries. In this guide you will only need three of them:

  • typed/2htdp/image deals with images.

  • typed/2htdp/universe deals with big-bang, keystroke handlers, clock-tick handlers, etc.

  • typed/test-engine/racket-tests deals with unit tests, e.g. check-expect and check-within.

In order to include these libraries inside your code, you need to use the "require" function. The syntax is simple:

  • (require library_name)

For instance, if you work with images in your code:

  • (require typed/2htdp/image)

If you want to import multiple libraries, you do not need to type "require" over and over:

  • (require typed/2htdp/image
         typed/2htdp/universe
         typed/test-engine/racket-tests)

You are all set up!
